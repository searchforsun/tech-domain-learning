# demo/README.md 写作指引

示例目录的入口文件，让读者知道「跑什么、怎么跑、跑完看什么」。

## 必须包含

| 要素 | 说明 |
|------|------|
| 总览表 | 文件 → 对应知识点 → 建议运行顺序 |
| 环境要求 | 语言版本、工具、依赖（精确版本号，不写「最新版」） |
| 运行命令 | 每个示例一条可复制命令；Windows PowerShell 差异单独标注 |
| 学习建议 | 先读哪个、改哪段代码、观察什么现象——引导做对比实验 |

## 模板

模板是技术无关的骨架，根据实际方向替换文件类型和工具：

```markdown
# 阶段 N 示例：<一句话主题>

（1–2 句：本目录示例验证什么，不演示什么。）

## 示例总览

| 入口文件 | 对应知识点 | 建议顺序 |
|----------|-----------|----------|
| `<入口文件>` | <知识点 1> | 先运行 |
| `<入口文件>` | <知识点 2> | 后运行 |

## 环境要求

- 运行时/语言：<精确版本>（如 MySQL 8.0+、JDK 17+、Python 3.10+）
- 工具：<精确版本>（如 Docker 24+、Maven 3.8+）
- 其他依赖：<数据库/中间件版本>

## 运行命令

### 示例 A

```bash
<可复制命令>
```

（如有平台差异，单独标注 PowerShell 命令）

## 学习建议

1. **示例A**：调整 `<某参数>`，观察 `<预期现象>`。
2. **示例B**：修改 `<某配置>`，对比 `<现象A>` 与 `<现象B>`。
3. 自问：<验证性问题>。

## 与 THEORY.md 的配合

先读理论稿「<某节>」，再跑代码；结果对照文中「<关键概念>」印证。
```

常见的 demo 形态：

| 技术方向 | demo 形态 | 环境依赖 |
|----------|-----------|----------|
| 数据库/SQL | `.sql` 脚本 + Docker Compose 启动实例 | Docker, mysql/pg CLI |
| Java 后端 | Maven/Gradle 单模块项目 | JDK, Maven |
| 消息队列 | Docker Compose（broker + producer/consumer） | Docker |
| 分布式系统 | Docker Compose 多服务 + 配置集 | Docker |
| 前端框架 | 组件文件/单页 | Node, npm |
| 运维/网络 | 配置文件集 + 启动脚本 | 对应运行时 |
