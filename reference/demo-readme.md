# demo/README.md 写作规范（必读）

| 要素 | 说明 | 必须程度 |
|------|------|----------|
| **示例总览表** | 每个文件/目录 → 对应知识点 → 建议操作顺序 | 必须 |

| **运行命令** | 每个示例的可复制命令（含平台差异说明） | 必须 |

## 写作要点

| 要点 | 说明 |
|------|------|
| 总览表列宽 | 文件名、知识点、顺序三列，简洁明了 |
| 环境要求 | 精确版本号，避免「最新版本」等模糊描述 |
| 运行命令 | 每个示例单独一个命令块；Windows PowerShell 差异单独说明 |
| 学习建议 | 引导读者做对比实验（改参数/换算法/观察现象），而非只是「运行看结果」 |
| 理论配合 | 指明先读哪一节的 THEORY.md，再跑哪个 demo，形成「理论 → 实验 → 印证」闭环 |
