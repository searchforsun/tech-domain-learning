# THEORY.md 写作规范（必读）

本文件是 `THEORY.md` 的**唯一细纲**，规定所有阶段理论文档的格式、内容与风格。

## 一、文件头部格式（必须）

每个阶段 `THEORY.md` 头部必须包含：

```markdown
# 阶段 N：<阶段标题>

> 与 [`ROADMAP.md`](../ROADMAP.md) 阶段 N 对齐。动手印证见本目录 [`demo/README.md`](demo/README.md)。**生成说明**：推荐阅读链接基于官方文档入口撰写；组件与默认值持续演进，**请以官方文档与团队规范为准**。

（正文开场段落：1–2 句承上启下，说明本阶段在整个路线中的位置与核心目标）
```

**要点**：
- `阶段 N：`必须与 ROADMAP.md 中的阶段序号和标题一致
- 头部使用 `>` 引用块，包含：ROADMAP 对齐声明、demo 入口路径、生成说明（官方文档为准）
- 开场段落一句话说明本阶段在路线中的位置（前置阶段承接 / 后续阶段铺垫）

---

## 二、本阶段知识地图（必须）

紧接文件头部之后，放置**知识地图表格**：

```markdown
## 本阶段知识地图

| 块 | 你要带走的抓手 |
|----|----------------|
| 一 | <核心主题 A> |
| 二 | <核心主题 B> |
| 三 | <核心主题 C> |
（根据实际节数调整，4–6 行为宜）
```

**何时需要**：主题数 ≥ 4 或章节较长时必须添加；2–3 节可省略。

---

## 三、路线要点 ↔ 本文章节（必须）

紧接知识地图之后：

```markdown
**路线要点 ↔ 本文章节**

| `ROADMAP.md` 阶段 N 要点 | 本文展开位置 |
|--------------------------|----------------|
| <路线中的核心主题 1> | **一** |
| <路线中的核心主题 2> | **二** |
| <路线中的核心主题 3> | **三** |
```

---

## 四、正文章节结构

### 4.1 分节原则

- `## 一、`、`## 二、`… 按序排列，标题用描述性短语（自解释）
- 节数从 ROADMAP.md 核心主题数推导，**禁止套固定骨架填空**
- 每 `##` 节须有实质性内容（论证 + 例证），禁止空壳节或纯列表节

### 4.2 节内结构（三层分工）

| 层 | 内容 | 约束 |
|----|------|------|
| **索引层** | 首段（动机 + 结论取向，≥3 句）+ 知识点表格（# \| 知识点 \| 抓住什么）或分条 | 不写长论证；不与论证层逐句重复 |
| **论证层** | `###` 小标题（**编号 + 描述**，如 `1.1 穿透：防御不存在的数据`）；段落用完整因果句；列表前一句总起、后一句收束 | 每 `##` 节 ≥2 个 `###`；**`###` 编号与 `##` 节号对应**（一节内从 1.1 起递增）；结论加粗后再展开 2–5 句 |
| **具象层** | 短例（嵌入「例如…」1–3 句）；对比表；Mermaid；点名 `demo/` 路径 | 每节至少 1 处具象化；长脚本放 `demo/` |

### 4.3 每节末尾（必须）

每个实质性 `##` 大节末尾，必须紧跟：

```markdown
---

**本节提要（延伸学习）**

- **核心概念**：<3–6 个关节短语，与正文及索引层术语一致>
- **拓展提问提示词**

> 主题：<本节完整主题>。核心概念：<写全所有核心概念>。请拓展：<范围与期望产出写清，不依赖会话/附件/上文等指代，不含身份铺垫>
```

**要点**：
- `---` 分割线前是正文收尾
- 「核心概念」3–6 个短语，用分号分隔
- 拓展提问提示词用 `>` 块，格式：主题 + 核心概念 + 请拓展（三段式）

---

## 五、文末推荐阅读

紧接最后一个 `##` 大节末尾提要之后：

```markdown
## 推荐阅读

> 说明在前、链接行仅 URL（复制时不会夹带额外字符）。

- **标题**
  - 关联主题：<本节涵盖的核心内容>
  - <完整 URL，WebSearch 验证>
  - 检索：`<检索关键词>`
```

**排版规则**：
- 说明文字在 URL 之前，URL 单独一行，该行**仅含 URL**（前后可留白，勿在同一行拼接其他字符）
- 检索关键词用反引号包裹

## 七、版式约束

| 规则 | 说明 |
|------|------|
| 标题层级 | `#` 仅文档标题；正文 `## 一、`… 顺排；`###` 用于节内小节 |
| 代码块 | 正文只放短片段（≤10 行）；长脚本放 `demo/` |
| 列表节奏 | 超过 4 条连续列表须拆分进表格或插入叙述性过渡 |
| 具象化 | 每节至少 1 处具象化（图/表/代码/例）；图前一句目的，图下 1–2 句收束 |
| 衔接 | 先定义再使用；小节过渡；重要概念带反例或边界 |

---

## 八、质量门禁自检（约 30 秒）

只看一级小标题 + 知识点表头 + 路线↔章节表头：

1. 能否说清本阶段的 3 条主线？
2. `ROADMAP.md` 每条核心主题是否都有正文展开？
3. 是否出现「整节只有表/列表」而无论证的情况？
4. 每节末尾是否有「核心概念 + 拓展提问提示词」？
5. 推荐阅读 URL 是否为真实可访问的官方文档？
