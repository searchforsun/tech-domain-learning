# 命名与目录布局

## 命名语言

| 类型 | 语言 | 示例 |
|------|------|------|
| 专题根路径 | 中文 | `learning/Java 分布式架构/` |
| 阶段文件夹 | 中文 | `01-分布式心智模型与故障优先设计/` |
| 标准文件名 | 英文 | `ROADMAP.md`、`PROGRESS.md`、`THEORY.md`、`README.md` |
| 代码文件名 | 英文（工具链惯例） | `RaceConditionDemo.java` |
| Markdown 标题与正文 | 中文 | 全文中文说明 |
| 代码注释 | 中文为主 | `// 模拟网络调用延迟` |

## 目录结构

```
learning/<技术专题中文简称>/
├── ROADMAP.md
├── PROGRESS.md
└── <序号>-<阶段中文名>/
    ├── THEORY.md
    └── demo/
        ├── README.md
        └── <示例文件/目录>
```

## 阶段文件夹布局

```
01-基础语法/
├── THEORY.md
└── demo/
    ├── README.md            # 必含：文件→知识点表、环境、运行命令、学习建议
    ├── 01-变量与类型/       # 可选：子主题中文文件夹
    └── ...
```

## 关键原则

- 阶段文件夹名 = `序号-阶段中文名`，与 ROADMAP.md 阶段名严格一致
- `THEORY.md` 是理论讲义，放 `demo/` 目录外
- `demo/README.md` 是示例入口，放 `demo/` 目录内
- `demo/` 下可按需创建子目录（如按子主题分组的示例）
