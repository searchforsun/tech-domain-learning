# tech-domain-learning

面向编程或任意技术领域的系统化学习。生成学习路线 ROADMAP.md、进度追踪 PROGRESS.md，按阶段交付理论讲义 THEORY.md 与可运行示例 demo/。支持学习过程中的增量问答补充与进度归档。

## 触发短语

帮我学 X · 制定 X 学习路线 · 生成 X 的阶段讲义 · 我想成为 X 架构师该怎么学 · 进入下一阶段 · 本阶段学完了

---

## 核心原则

1. **知识来源**：稳定（官方文档/权威出版物）、可信（可追溯）、真实（WebSearch 验证 URL）、前沿（主流稳定版本）
2. **单一事实来源**：`PROGRESS.md` 是唯一进度真相；阶段文件夹名与路线中的阶段名保持一致（`序号-阶段中文名`）
3. **审阅门禁**：交付 `ROADMAP.md` 与初始 `PROGRESS.md` 后，**暂停并请用户审阅**；确认前不生成阶段 1 的完整物料
4. **理论质量**：`THEORY.md` 以 `reference/theory.md` 为唯一细纲；`demo/README.md` 以 `reference/demo-readme.md` 为唯一细纲

## 目录结构

```
learning/<技术专题中文简称>/
├── ROADMAP.md          # 完整路线（阶段、目标、产出、依赖顺序）
├── PROGRESS.md         # 当前阶段、历史记录、阶段清单
└── <序号>-<阶段中文名>/
    ├── THEORY.md       # 理论讲义（严格按 reference/theory.md）
    └── demo/           # 可运行示例
        └── README.md   # 必含：文件→知识点表、环境、运行命令、学习建议
```

详细布局见 [`reference/naming-and-layout.md`](reference/naming-and-layout.md)。

## 工作流 A：初始化路线

| 步骤 | 动作 | 产出检查 |
|------|------|----------|
| 1 | 确认技术方向，确定专题根目录 `learning/<专题中文简称>/` | 目录名简短中文、无歧义 |
| 2 | **WebSearch** 检索该技术栈「当前生态要点」：版本、官方文档入口、主流工具链 | 检索结果摘要（版本号、文档首页 URL） |
| 3 | 编写 **`ROADMAP.md`**：4–8 阶段 + ASCII 箭头图 + 路线总览表 + 各阶段详情 | 模板见 `reference/roadmap-progress.md` |
| 4 | 初始化 **`PROGRESS.md`**：当前阶段标为「未开始」，阶段清单表与路线一致，完成记录为空 | 两文件阶段数量、名称一一对应 |
| 5 | **暂停并交付**：请用户阅读路线、调整阶段划分或深度、**询问是否确认开始** | — |
| 6 | 用户确认后 → 进入工作流 B，从阶段 1 开始 | — |

## 工作流 B：生成某一阶段内容

**前置**：读取 `PROGRESS.md` 确认当前阶段；若用户说「下一阶段」，则取**第一个未完成**的阶段。

| 步骤 | 动作 | 产出检查 |
|------|------|----------|
| 1 | 创建阶段文件夹 `learning/<专题>/<序号>-<阶段中文名>/` | 名称与 `ROADMAP.md` 一致 |
| 2 | 撰写 **`THEORY.md`**：严格按 `reference/theory.md`；文末推荐阅读按 `reference/theory-recommended-reading.md`，**WebSearch 后写入真实 URL** | 每节含知识点索引表 + `###` 论证 + 具象层 + 核心概念与拓展提问提示词 |
| 3 | 创建 **`demo/`**：可运行示例 + **`README.md`**（必含） | `README.md` 满足 `reference/demo-readme.md` |
| 4 | 代码加适量中文注释（说明复杂处即可，不必逐行） | — |
| 5 | 将 `PROGRESS.md` 中本阶段状态更新为 **进行中** | — |

## 工作流 C：学习中的问答与补充

用户针对**当前阶段**提问或要求补充时：

1. 以 `PROGRESS.md` 定位当前阶段文件夹
2. 在 **`THEORY.md`** 中追加或修订：新增 `##` 大节须同步补全末尾「核心概念 + 拓展提问提示词」；**永不覆盖用户手写笔记**
3. 在 **`demo/`** 中补充注释或重构；若新增文件或改变学习顺序，**同步更新 `demo/README.md`**

---

## 工作流 D：阶段完成后的进度更新

| 步骤 | 动作 |
|------|------|
| 1 | 更新 `PROGRESS.md` 阶段清单表：状态 → **已完成**，填写完成日期与学习时长 |
| 2 | 在「阶段完成记录」追加一条：含日期、时长、**简要收获**（2–5 条，与用户协作） |
| 3 | 推进**当前阶段**到下一阶段，标记为「未开始」 |
| 4 | **提示用户**：下一阶段文件夹名 + 入口文件（`THEORY.md` + `demo/README.md`） |

详细模板见 `reference/roadmap-progress.md`。

## 质量门禁

生成或修改内容后自检以下 6 条：

| # | 检查项 | 正确做法 | 常见违规 |
|---|--------|----------|----------|
| 1 | **时效标记** | 路线含生成日期 +「请以官方文档为准」免责 | 无日期声明 |
| 2 | **路线图** | 自上而下 ASCII 箭头图，阶段 4–8 个 | 缺图或超过 8 阶段未归类 |
| 3 | **反提纲** | 每 `##` 节有连续因果叙述，不可整节只有表/列表 | 纯列表节、索引表替代论证 |
| 4 | **版式一致** | `## 一、`… 顺排；每节末含核心概念 + 拓展提问提示词 | 多级标题骨架雷同、缺提要 |
| 5 | **知识来源** | 所有 URL/API/版本号经 WebSearch 验证，引用源自官方文档或权威出版物 | 编造 URL、引用低质量来源、使用已废弃方案 |
| 6 | **URL 真实** | 推荐阅读链接经 WebSearch 验证，链接行仅 URL | 编造子路径、URL 后拼接中文 |

## 常见问题

**Q：用户想跳过某些阶段怎么办？**
按用户要求调整 `ROADMAP.md` 和 `PROGRESS.md`，保持一致。

**Q：用户说"我已经会了阶段 1–2，从 3 开始"？**
读取现有 `PROGRESS.md`，将阶段 1–2 标记为已完成（日期写当前，备注"用户自评已掌握"），当前阶段设为阶段 3。

**Q：用户对路线有异议（增/删/改阶段）？**
在对话中讨论调整方案，达成一致后更新 `ROADMAP.md` 和 `PROGRESS.md` 的阶段清单表。

**Q：`demo/` 中放什么类型的代码？**
取决于技术方向：Java 后端用 Maven 项目；前端用组件文件；消息队列用 Docker Compose + 客户端示例；数据库用 SQL 脚本。原则：**最小化、可运行、可修改做实验**。
